# 《轻宠·领主 3D》需求文档（PRD）

文档版本：v1.1（内存指标放宽版）  
创建日期：2026-03-04  
文档状态：可执行初版（可直接进入排期）  
适用范围：MVP + 迭代版

## 1. 产品核心定位

一句话定位：  
国内首款「桌面3D悬浮宠 + LBS现实流浪宠物收留 + 五行回合制对战 + 现实区域占领」的虚实结合治愈系应用，主打低压力陪伴、轻量竞技、现实探索。

核心差异壁垒：
1. 桌面常驻陪伴、现实探索捕捉、轻量社交对战三位一体闭环。
2. 无强制任务、无数值碾压付费、无高社交压力。
3. 以情绪价值为入口，以轻竞技与现实探索形成长期留存。

## 2. 目标用户与场景

目标用户：
1. 18-35 岁城市用户，存在陪伴需求但排斥高负担社交。
2. 通勤与办公室场景用户，接受轻交互、短时玩法。
3. 喜欢收集与地图探索，但不愿投入重度 MMO 级时间成本的用户。

核心使用场景：
1. 工作学习时，宠物静默陪伴并做低频反馈。
2. 通勤/散步中，用户在真实地理位置收留流浪宠物。
3. 零碎时间内完成 1v1 回合对战并争夺现实区域收益。
4. 跨端同步，PC 陪伴与手机探索互相促进。

## 3. 产品目标与北极星指标

业务目标：
1. 建立“陪伴 -> 探索 -> 对战 -> 占领 -> 反哺探索”的可持续循环。
2. 形成低打扰、低门槛、高黏性的日常使用习惯。
3. 在不依赖数值付费的前提下验证玩法留存与区域活跃度。

核心指标（MVP 验收）：
1. D1 留存 >= 35%。
2. 次周活跃用户中，完成一次完整玩法闭环比例 >= 40%。
3. 单次对战同步延迟 p95 < 150ms。
4. LBS 收留触发精度误差 p95 < 50m。
5. PC 空闲 CPU < 8%，内存 < 150MB（单宠物实例）。
6. 崩溃率 < 0.1%。

## 4. 范围定义

MVP（优先交付）：
1. PC 端 3D 桌面悬浮宠基础能力。
2. LBS 流浪宠物刷新与收留。
3. 五行回合制 1v1 对战引擎与实时同步。
4. 现实区域占领最小可用版。
5. 情绪镜像基础版（静默反馈）。
6. 基础跨端账号与数据同步。

迭代版（次优先级）：
1. 区域占领玩法深化（榜单、驻守、收益可视化）。
2. 轻社交（微光相遇、匿名守护、私密约战）。
3. 移动端悬浮窗/小组件强化。
4. 养成系统（等级、亲密度、换装）补齐。

当前不做（非范围）：
1. 强社交公会体系。
2. 开放世界地图与大型实时多人同屏。
3. 重剧情分支系统与高制作过场剧情。

## 5. 最终技术选型（锁定）

| 模块 | 技术选型 | 选择理由 |
|------|----------|----------|
| PC 桌面端 | Electron 31 + React 19 + React Three Fiber（Three.js r162） | 支持透明无边框窗口、鼠标穿透、GLB 渲染与按需降帧 |
| 移动端 | Flutter 4.0 + Unity 6000 LTS（URP） | UI 与 3D 分工清晰，跨端一致性和性能平衡较好 |
| 后端/数据 | Supabase（PostgreSQL 16 + PostGIS 3.4 + Realtime Broadcast） | LBS 地理能力+实时同步+较低运维成本 |
| 对战引擎 | TypeScript 状态机 + Supabase Realtime | 回合制可前后端协同校验，防作弊实现成本可控 |
| LBS 定位 | 国内腾讯位置 SDK 7.6+；海外 Google Maps SDK | 国内外定位覆盖与精度平衡 |
| 资源方案 | GLB/GLTF + Rive 动画 | 3D 角色表现与轻量 UI 动效兼顾 |

## 6. 核心玩法闭环

闭环定义：  
现实 LBS 收留流浪宠物 -> 桌面养成解锁属性/技能 -> 1v1 回合对战 -> 占领现实区域 -> 该区域捕捉收益翻倍 -> 继续探索。

闭环验收口径：
1. 用户可在单账号单日内完成完整闭环至少 1 次。
2. 每个环节均存在可观测事件埋点，且可串联到同一会话。
3. 闭环中断时可恢复，不出现永久卡死状态。

## 7. 功能需求（FR）

### 7.1 桌面 3D 悬浮宠

FR-001：支持透明无边框窗口、置顶、拖拽、缩放。  
FR-002：支持动态鼠标穿透，仅模型区域可交互。  
FR-003：支持 Idle/Attack/Hit/Dodge 等基础动画状态切换。  
FR-004：宠物静止时自动降帧至 10fps，交互时恢复至 60fps。  
FR-005：支持开机自启与托盘控制（显示/隐藏/退出）。

验收标准：
1. 透明区域点击不拦截底层应用，误拦截率 < 1%。
2. 连续运行 8 小时不出现窗口黑边、闪烁或明显内存泄漏。
3. PC 端性能满足生死线。

### 7.2 LBS 流浪宠物收留

FR-006：地图点位按小时刷新流浪宠物。  
FR-007：用户进入 100m 范围内可触发“一键收留”。  
FR-008：按地标类型映射宠物属性稀有池。  
FR-009：收留后生成“记忆标签”并永久保留于图鉴。  
FR-010：应用最小化时可触发点位提醒（可关闭）。

验收标准：
1. 地理围栏触发误差 p95 < 50m。
2. 地标到属性映射准确率 100%。
3. 收留成功后图鉴、属性、记忆标签三方数据一致。

### 7.3 五行回合制对战引擎

FR-011：实现五行克制链：金->木->土->水->火->金。  
FR-012：克制增伤 1.5 倍，被克制衰减 0.7 倍。  
FR-013：实现战术三角：普攻打断属性攻击；属性攻击无视闪避；闪避躲过普攻。  
FR-014：实现异常状态：火灼烧、水冻结、木寄生、金易伤、土石化。  
FR-015：实现怒气机制，满足阈值后释放必中高伤大招。  
FR-016：房间创建、匹配、回合同步、断线重连。  
FR-017：服务器权威校验，非法指令拦截并记录。

验收标准：
1. 规则计算单测通过率 100%，关键公式误差 0。
2. 对战同步延迟 p95 < 150ms，房间恢复成功率 >= 99%。
3. 非法操作拦截率 100%（在测试脚本覆盖范围内）。

### 7.4 现实区域占领

FR-018：支持区域入侵、驻守、占领状态流转。  
FR-019：占领后触发区域捕捉收益翻倍。  
FR-020：每日免费入侵次数默认 2 次（可配置）。  
FR-021：展示区域排行、占领记录、区域归属光环。

验收标准：
1. 对战胜负与占领归属一致率 100%。
2. 收益加成立即生效，跨端同步延迟 < 200ms。

### 7.5 轻社交系统

FR-022：微光相遇，同点位宠物自动互动且不暴露用户身份。  
FR-023：匿名守护，仅反馈“有温柔的人守护了你”。  
FR-024：支持好友与私密约战（迭代版）。

验收标准：
1. 微光互动触发成功率 >= 99%。
2. 匿名场景下无可逆身份信息泄露。

### 7.6 情绪镜像系统

FR-025：PC 侧采集打字速度、活跃时长、时段信息（仅行为统计）。  
FR-026：移动侧采集点击频率、使用时长、作息节律。  
FR-027：映射为宠物行为反馈，不使用强打断弹窗。  
FR-028：支持每日情绪小结生成。

验收标准：
1. 行为采集需最小化与脱敏，不采集输入内容本身。
2. 反馈触发节奏合理，无连续干扰投诉型触发。

## 8. 非功能需求（NFR）

NFR-001：PC 内存 p95 < 150MB（单宠物实例，空闲场景）。  
NFR-002：移动端内存 p95 < 220MB（Flutter + Unity 前台场景）。  
NFR-003：API 响应 p95 < 100ms。  
NFR-004：对战同步延迟 p95 < 150ms。  
NFR-005：定位触发精度误差 p95 < 50m。  
NFR-006：稳定性崩溃率 < 0.1%。  
NFR-007：关键路径必须有埋点、日志、报警。  
NFR-008：敏感数据最小化存储，导出日志需脱敏。

## 9. 核心数据模型（概要）

关键实体：
1. `users`：用户基础资料、隐私设置、设备绑定。
2. `pets`：宠物实例、属性、技能、情绪状态、归属用户。
3. `pet_memory_tags`：收留记忆标签。
4. `geo_spawn_points`：流浪宠物刷新点位（PostGIS）。
5. `captures`：收留记录。
6. `battle_rooms`：对战房间元数据与状态。
7. `battle_round_actions`：每回合操作与校验结果。
8. `territories`：区域归属、驻守信息、收益系数。
9. `territory_logs`：占领与收益变更流水。
10. `emotion_snapshots`：情绪镜像输入与输出快照。

## 10. 埋点与观测

必须埋点事件：
1. `pet_window_interact`
2. `lbs_capture_attempt` / `lbs_capture_success`
3. `battle_room_created` / `battle_round_synced` / `battle_finished`
4. `territory_invade` / `territory_occupied`
5. `emotion_feedback_triggered`
6. `client_crash` / `render_fallback`

报表维度：
1. 闭环完成率（日/周）。
2. 各环节流失率与平均耗时。
3. 设备性能分层下的稳定性指标。

## 11. 安全与合规

1. 默认关闭精确定位后台常驻，首次触发时显式授权。
2. 情绪镜像仅采行为统计，不采集文本内容与隐私正文。
3. 对战与占领相关写操作必须经过服务端签名校验。
4. 日志导出采用匿名化策略，移除手机号、邮箱、精确坐标。

## 12. 成本与预算基线

预算目标：100 美元开发期预算可覆盖 MVP 构建与初期验证。  
结构基线：
1. 大模型辅助开发成本约 10 美元量级（按你给出的 80 万 Token 预算）。
2. Supabase Pro 约 25 美元/月。
3. 地图 SDK 使用免费额度覆盖早期阶段。

说明：实际成本受 MAU、定位调用频率、实时频道峰值影响，需按周复盘。

## 13. 风险与规避

1. 鼠标穿透兼容风险。  
规避：`mousemove + setIgnoreMouseEvents` 方案 + 多 DPI/多屏矩阵测试。
2. 移动端 3D 发热风险。  
规避：URP 轻量化、阴影降级、后台降帧至 10fps。
3. 定位漂移风险。  
规避：卡尔曼滤波 + 围栏容错 + 室内外定位模式切换。
4. 对战不同步风险。  
规避：锁步回合制 + 服务端权威修正 + 断线重连恢复。
5. 地理查询性能风险。  
规避：PostGIS 索引、热点缓存、分页限制与查询限流。

## 14. 发布准入（Go/No-Go）

必须同时满足：
1. 核心闭环全链路通过率 >= 98%。
2. P0/P1 缺陷清零。
3. 生死线性能指标全部达标。
4. 安装包可用且跨端数据同步稳定。
5. 关键埋点完整，数据可回收可分析。
