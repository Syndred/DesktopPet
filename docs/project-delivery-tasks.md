## 1. 交付定义（Definition of Delivery）

满足以下条件，才算“可运行交付”：

1. 代码可在新环境一键安装并启动核心端（Desktop）。
2. 核心玩法闭环可跑通：收留 -> 对战 -> 占领 -> 收益回流。
3. 测试分层可执行并通过（unit/integration/e2e/performance/security）。
4. 发布演练、回滚演练、灾备演练有脚本且可复现。
5. 交付文档齐全：运行配置、测试策略、发布/回滚手册、灾备手册、任务进度。

## 3. 交付级任务清单（每项含测试与验收）

### D-001 工程底座与仓库规范
状态：✅ 完成

任务内容：
1. Monorepo 目录与 workspace 配置。
2. 统一 TypeScript 配置、Lint、Typecheck、测试命令。
3. 环境变量模板与基础文档。

测试要求：
1. `npm run lint` 通过。
2. `npm run typecheck` 通过。
3. 新环境执行 `npm install` 无阻塞错误。

验收标准：
1. 首次克隆仓库可完成依赖安装。
2. 质量门禁命令可一键执行。

### D-002 共享协议与领域模型
状态：✅ 完成

任务内容：
1. shared-types：战斗、事件、契约、错误码。

测试要求：
1. 契约单测覆盖关键接口。
2. 错误码映射测试通过。

验收标准：
1. 跨模块不出现字段语义冲突。
2. 关键输入输出可被自动化校验。

### D-003 数据层与安全策略（Supabase/RLS）
状态：✅ 完成（以本地模拟 + 迁移脚本验证为主）

测试要求：
1. 迁移脚本可重复执行。

验收标准：
1. 核心查询路径具备稳定返回。
2. 安全策略可通过自动化验证。

### D-004 PC 桌宠运行时（透明窗/托盘/穿透）

## 4. 统一验证命令（交付验收入口）

1. 安装依赖：`npm install`
2. 质量门禁：`npm run verify`
3. PC 运行时冒烟：`npm run pc:smoke`
4. 启动桌宠：`npm run dev:pc`
5. 发布演练：`npm run release:prepare`
6. 回滚演练：`npm run release:rollback -- --target=<version>`
7. 灾备演练：`npm run dr:backup -- --label=<label>` 与 `npm run dr:restore -- --label=<label>`
