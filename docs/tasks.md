## 4. 待办任务总览（Backlog）

### G2 核心能力补完

#### T-201 宠物仓库持久化（Supabase）[PC端已完成本地持久化与接口接入]
- 目标：将本地 `PET_ROSTER` 迁移为后端持久化数据。
- 开发内容：
  1. 新建 `pets`、`pet_instances`、`pet_capture_records` 表及索引。
  2. 增加宠物查询、设为出战、收养入库 API。
  3. PC 端仓库改为实时读取后端数据。
- 测试用例：
  1. 首次登录自动拉取仓库数据。
  2. 设为出战后重启应用仍保持。
  3. 并发写入不产生重复宠物实例。
- 验收标准：
  1. 出战宠物、收养时间、序号在重启后保持一致。
  2. 关键 API 响应 `p95 < 100ms`。

#### T-202 对战结果持久化与战报 [PC端已完成]
- 目标：每场对战可追溯、可统计。
- 开发内容：
  1. 新建 `battle_sessions`、`battle_round_events`。
  2. 记录动作、伤害、状态变化、结算结果。
  3. 面板新增最近战报列表。
- 测试用例：
  1. 战斗结束后自动写入一条完整会话。
  2. 会话内回合事件数量与实际回合数一致。
- 验收标准：
  1. 战报查询按时间倒序正确。
  2. 不出现缺回合、缺结算数据。

### G3 地图收养玩法（新增）

#### M-301 地图基础能力接入 [DONE-PC演示版]
- 目标：在 PC（演示）与移动端（正式）建立统一地图能力。
- 开发内容：
  1. 封装地图 Provider 抽象层（腾讯/谷歌）。
  2. 统一经纬度、坐标系与权限状态结构。
  3. 提供 `getCurrentLocation`、`watchLocation`、`distanceTo` 基础接口。
- 测试用例：
  1. 定位权限拒绝、允许、系统关闭三种场景可区分。
  2. 连续定位更新无崩溃且字段完整。
- 验收标准：
  1. 定位状态机完整可用。
  2. 各端统一接口可直接驱动上层业务。

#### M-302 LBS 点位刷新服务 [DONE-PC演示版]
- 目标：实现按地标规则刷新“流浪宠物点位”。
- 开发内容：
  1. 新建 `wild_pet_spawns`、`landmark_zones` 数据结构。
  2. 实现按小时刷新、按地标映射属性与稀有度。
  3. 提供“当前位置附近点位”查询 API。
- 测试用例：
  1. 刷新周期正确（跨小时更新）。
  2. 地标到属性映射命中率为 100%。
  3. 稀有度抽样分布符合配置误差范围。
- 验收标准：
  1. 100m 范围查询稳定返回。
  2. 点位刷新与查询无明显延迟抖动。

#### M-303 地图展示与交互层
- 目标：可在地图上看到并筛选可收养点位。
- 开发内容：
  1. 地图 marker 渲染、聚合与详情卡片。
  2. 按属性/稀有度筛选。
  3. 点位状态（可收养/已收养/冷却）可视化。
- 测试用例：
  1. 100+ marker 场景渲染不卡顿。
  2. 筛选条件组合时结果正确。
- 验收标准：
  1. 点位可视化清晰，无重复 marker。
  2. 交互响应流畅，无误触发。

#### M-304 收养流程联动 [DONE-PC演示版]
- 目标：从地图点位到宠物入库形成完整闭环。
- 开发内容：
  1. 进入范围后激活“收养”操作。
  2. 完成收养后写入仓库、图鉴、记忆标签。
  3. 点位状态立即更新并进入冷却。
- 测试用例：
  1. 不在范围内无法收养。
  2. 收养成功后仓库与图鉴均可查。
  3. 重复收养同点位被正确拦截。
- 验收标准：
  1. 收养事务一致性 100%。
  2. 收养后反馈完整，无脏数据。

#### M-305 地图与区域占领收益联动
- 目标：占领区域对地图收养收益产生实时加成。
- 开发内容：
  1. 将区域归属映射到地理范围。
  2. 在地图点位上标注当前收益倍率。
  3. 结算时应用倍率并落库。
- 测试用例：
  1. 占领成功后对应区域倍率即时变化。
  2. 跨区域移动时倍率自动切换。
- 验收标准：
  1. 地图展示倍率与结算倍率一致。
  2. 不出现跨区错算。

#### M-306 后台提醒与地理围栏
- 目标：应用最小化后仍可收到附近可收养提醒。
- 开发内容：
  1. 地理围栏触发策略与去抖。
  2. 冷却期点位不重复提醒。
  3. 提醒文案与频率节流。
- 测试用例：
  1. 连续进出围栏不会重复轰炸提醒。
  2. 冷却点位不再触发。
- 验收标准：
  1. 提醒准确且低打扰。
  2. 不影响基础性能指标。

#### M-307 地图模块验收包
- 目标：形成可交付的地图功能验收材料。
- 开发内容：
  1. 地图模块测试报告（功能/性能/异常）。
  2. 演示脚本：定位 -> 点位 -> 收养 -> 入库。
  3. 故障回滚预案（地图 SDK/定位权限异常）。
- 测试用例：
  1. 完整 E2E 跑通。
  2. 关键异常场景有回退路径。
- 验收标准：
  1. 地图收养闭环可重复演示。
  2. 文档与实现一致。
