# 灵境 PC 版配置与使用说明

文档版本：v1.0  
更新时间：2026-03-07

## 1. 目标

本说明用于你当前这版 PC 客户端（双开窗口联机实测）：

1. 启动独立登录窗口。
2. A/B 两个账号互发对战申请并接受。
3. 自动匹配房间并进入联机对战。
4. 需要时可用“高级选项”手动输入房间号兜底。

## 2. 你需要准备哪些数据

联机最少需要这 3 个配置项：

1. `SUPABASE_URL`
2. `SUPABASE_ANON_KEY`
3. `SUPABASE_DUEL_FUNCTION`（默认 `duel-online`）

推荐放在：

`scripts/desktop/local-online-env.cmd`

示例：

```cmd
@echo off
set SUPABASE_URL=http://你的联机服务地址:8000
set SUPABASE_ANON_KEY=你的匿名公钥
set SUPABASE_DUEL_FUNCTION=duel-online
```

## 3. 这些数据从哪里获得

### 3.1 `SUPABASE_URL`

如果你是自建部署：

1. 使用你对外暴露的 API 网关地址（例如反向代理域名或公网 IP+端口）。
2. 必须保证客户端机器可访问该地址。

常见形式：

1. `http://<公网IP>:8000`
2. `https://<你的域名>`

### 3.2 `SUPABASE_ANON_KEY`

从你部署时生成的 Supabase 配置里取匿名公钥（anon key）。  
不要填 service role key（那是服务端密钥，不应放客户端）。

### 3.3 `SUPABASE_DUEL_FUNCTION`

当前对战结算函数名，默认：

`duel-online`

如果你在服务端改了函数名，这里要同步改成同名。

## 4. 本地安装与启动

项目根目录执行：

```powershell
npm install
```

单开调试：

```powershell
npm run dev:pc
```

双开联机（推荐）：

```cmd
scripts\desktop\start-online-dual.cmd
```

也可以分开开两个窗口：

1. `scripts\desktop\start-online-a.cmd`
2. `scripts\desktop\start-online-b.cmd`

## 5. GUI 双开联机实测流程

1. 在 A 窗口注册/登录账号 A。
2. 在 B 窗口注册/登录账号 B。
3. A 在“对战申请”里搜索 B 的账号并发起申请。
4. B 收到申请后点击“接受”。
5. 接受后系统会自动建房并同步房间号，双方进入联机对战状态。
6. 点击“开始对战”进入实时战斗。

说明：

1. 正常流程不需要手动输入房间号。
2. 手动房间号已移动到“高级选项（默认折叠）”，只作为兜底。

## 6. 高级选项（手动房间）怎么用

在“对战申请”分组中：

1. 点“高级选项：手动房间控制”展开。
2. 可手动创建房间、输入房间号加入、离开房间。
3. 折叠/展开状态会记忆，下次打开保持上次状态。

适用场景：

1. 自动匹配链路异常时。
2. 你要强制让某端加入指定房间时。

## 7. 账号与数据隔离（双开关键）

双开脚本会给 A/B 注入不同的用户数据目录，避免两个窗口共用同一会话。

你会看到两个窗口可以同时登录不同账号。  
如果发现“两个窗口总是同一个账号”，通常是你没通过双开脚本启动。

## 8. 常见问题排查

### 8.1 显示“未配置 SUPABASE_URL/SUPABASE_ANON_KEY”

检查：

1. `local-online-env.cmd` 是否存在。
2. 三个变量名是否拼写正确。
3. 启动方式是否走 `start-online-*.cmd` 脚本。

### 8.2 发起申请后对方收不到

检查：

1. 两端是否都已登录且不是同一账号。
2. 服务端是否可访问（URL 可达）。
3. 防火墙/安全组端口是否已放行（你的网关/反代端口）。
4. 服务端日志里是否有 duel request / realtime channel 订阅记录。

### 8.3 接受申请后一方没进战斗

处理顺序：

1. 先看“联机状态”是否变为 active/waiting。
2. 若状态正常但没进战斗，点“开始对战”手动进入。
3. 若状态不同步，用“高级选项”手动加入同一房间号兜底。

### 8.4 放逐没有反应

当前版本是二次确认弹窗：

1. 先在仓库选中非出战灵宠。
2. 点“放逐”，会弹确认框（含编号、收留时间）。
3. 确认后才真正放逐。

### 8.5 快捷“出战”按钮看不到

当前版本每个非出战灵宠头像下方都有“出战”按钮。  
点击后会切换出战并直接关闭面板，回到桌宠展示。

## 9. 打包 EXE

按现有脚本执行打包（以项目脚本为准）：

```powershell
npm run build:pc
```

如果你沿用之前产物目录，常见输出位置：

`artifacts/pc-exe/`

## 10. 建议的上线前检查

1. 双开 10 轮邀请/接受回归，确认双方都能入战斗。
2. 断网重连后验证房间状态同步。
3. 中文/英文切换时仅显示当前语言文案。
4. 仓库操作（快捷出战、详情关闭、放逐确认）全链路回归。
